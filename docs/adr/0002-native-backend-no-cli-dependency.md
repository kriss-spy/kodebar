# Native backend — no upstream CLI dependency

The original v1 design wrapped the upstream `codexbar` CLI (spawning it per-provider). The redesign implements all provider probes natively in Rust, reading on-disk credentials directly. This was driven by three factors: (1) the user switched from Codex/Claude to OpenCode + Antigravity, and there is no equivalent upstream CLI for the OpenCode ecosystem that works on Linux — opencode-bar is macOS-only Swift; (2) spawning a subprocess per probe is inherently fragile (RPC errors, binary version drift, distro packaging issues); (3) the credential files (`~/.gemini/oauth_creds.json`, OpenCode `auth.json`, the dashboard auth cookie) are already on disk and well-documented by the reverse-engineering community, making native probing strictly simpler than wrapping a CLI that doesn't exist for this provider set.
